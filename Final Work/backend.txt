Backend на json файлах.

Сегодня будет еще доработан, в части постраничности - вывод общего количества страниц.

Описание бэкэнда:
Все запросы делаем через GET (чтобы не усложнять и в этом не путаться)
Все запросы отдают HTTP 404 в случае, если что-то не так (например, нет книги с таким id, или данных недостаточно - не переданы данные, перечисленные в "обязательно ждет")

book.get.php
отвечает данными одной книги
ОБЯЗАТЕЛЬНО ждет:
id (идентификатор книги)

пример ответа:
{
"id":1,
"title":"Программирование в 1С:Предприятие 8.3",
"publisher":"Питер",
"description":"...",
"picture":"1011878859.jpg",
"taken":false,
"taken_by":null
}

books.get.php
отвечает списком книг и количеством страниц
ОПЦИОНАЛЬНО ждет:
page (номер страницы) - если не передан, то бекэнд ответит всеми книгами (количество страниц при этом, все же, будет передано)

пример ответа:
{
books: [{
"id":1,
"title":"Программирование в 1С:Предприятие 8.3",
"publisher":"Питер",
"description":"...",
"picture":"1011878859.jpg",
"taken":false,
"taken_by":null
}],
total_pages: 5
}

taken_by - это идентификатор пользователя, взявшего книгу, taken - поле, где хранится дата, до которой книга взята, или false, если книгу можно взять.

book.set.php
установка статуса книги - это ручка для того чтобы взять или вернуть книгу. чтобы книгу взять - передаем take_till, а если надо вернуть книгу - не передаем этот параметр
ответ - true, если книгу успешно взяли/вернули, или HTTP 404 если что-то пошло не так
ОБЯЗАТЕЛЬНО ждет:
id (идентификатор книги)
user (идентификатор пользователя)
ОПЦИОНАЛЬНО ждет:
take_till (дата, до которой берем книгу) - если не передан, то будет выставлен false - книгу возвращаем

пример ответа:
true

reader.get.php
отвечает данными одного пользователя
ОБЯЗАТЕЛЬНО ждет:
id (идентификатор пользователя)

пример ответа:
{
"id": 1,
"name": "Юрий Гагарин"
}
