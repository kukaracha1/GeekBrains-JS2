Домашнее задание:

Библиотека
Представьте себе библиотеку организации, которая представляет собой шкаф с книгами и небольшой компьютер («библиотекарь»), на котором установлено ваше приложение. Приложение работает на веб-технологиях, что является дополнительным плюсом - можно работать с ним удаленно.

Сущности, с которыми работаем - книги и читатели.

Читатель - это имя и идентификатор (id - число, выдается при приеме на работу, вшивается в бейджик, который считывает библиотекарь, также id можно вводить клавиатурой, номер написан на бейджике).

Каждая книга - это id, название, издательство, аннотация, картинка обложки. Также у книги будет присутствовать состояние «взятости», которое либо false, либо дата, до которой взята книга (упрощение - вместо даты можно просто true)

Интерфейс читателя: Список книг, с постраничностью, с кнопками «подробнее» и «взять». При взятии интерфейс попросит номер читателя - вводится с клавиатуры, либо считывающим устройством, которое работает как клавиатура, после того как нему поднесен бейдж. При нажатии кнопки подробнее оказываемся на странице книжки, откуда ее тоже можно «взять».
Кнопка «взять книгу по номеру» - если книга уже в руках, «библиотекарь» позволяет считать ее номер (id) по штрихкоду, чтобы не заморачиваться с поиском книги в списке, и затем, также спросит номер читателя.
При взятии книги следует запросить дату, до которой книга берется, по умолчанию следует брать книгу на неделю.
Кнопка «мои книги» - показывает взятых читателем книг. Перед отображением книг спрашивается номер.
Кнопка «вернуть книгу» - позволяет вернуть книгу по ее номеру. Также кнопки «вернуть» присутствуют в списке моих книг - но там уже не будет спрошен номер повторно.

В перспективе, такой интерфейс позволяет создать второе рабочее место - администратора, где можно посмотреть список «должников» по книгам, добавить новую книгу, и т.п. Но зачет будет для роли читателя.

В идеале - покрыть это тестами.
